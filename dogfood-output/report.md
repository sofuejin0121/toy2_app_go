# agent-browser 動作確認レポート

実施日: 2026-04-19  
ツール: `agent-browser`（セッション `chirp-netlify-qa` / `chirp-local-qa`）

## 対象

| 環境 | URL | 備考 |
|------|-----|------|
| 本番 | `https://chirp-go.netlify.app` | ログイン資格情報なし（ゲスト・失敗ログイン中心） |
| ローカル | `http://127.0.0.1:5173` | `go run` + `vite`、DB `db/toy.db` |

## 本番（Netlify）で実施したこと

1. **トップ `/`** … ゲスト向け「Chirp へようこそ」表示を確認（`#root` innerText）。
2. **リロード** … 同文言のまま再表示。
3. **`/login`** … 不正メール＋パスワードで送信 → API メッセージ「メールアドレスまたはパスワードが正しくありません」を確認。
4. **`/signup`** … フォーム文言（名前・メール・パスワード等）を確認。
5. **`/users`（要ログイン）** … `/login` へリダイレクトを確認（`get url` が `/login`）。
6. **`/users/1`** … 「ユーザーが見つかりません」（本番 DB に id=1 がいない or 条件不一致）。**仕様／データ依存**。
7. **`/password_resets/new`** … パスワード忘れフォーム表示を確認。
8. **ネットワークログ**（`network requests --filter api`）… ゲスト時の `GET /api/me` 401、失敗ログインの POST 401 等を確認。

スクリーンショット: `dogfood-output/screenshots/01-home-guest.png`, `02-login-invalid.png`

## ローカル（同一オリジン + 既知ユーザー）で実施したこと

認証情報（DB 既存）: `user-1@example.com` / `password`

1. **ゲスト `/`** … 本番と同様のウェルカム表示。
2. **ログイン** … 上記で成功、`/users/2` へ遷移・プロフィール一覧表示を確認。
3. **ログイン後 `/`** … フィード・サイドバー（microposts / following 等）表示。**「認証が必要です」「ログインしていません」等の文言は全文 grep でヒットなし**。
4. **`/users/2` → すぐ `/`** … 同上、エラー文言なし。
5. **`/` リロード（ログイン維持）** … 同上。
6. **`/users`（一覧）** … ユーザー一覧・ページネーション表示。

スクリーンショット: `dogfood-output/screenshots/local-home-logged-in.png`

## 未実施・限界

- **本番でのログイン後ホーム／フィード** … 共有アカウントがないため未実施。
- **投稿・いいね・通知・設定の CRUD** … ローカルでは一覧・ホーム表示まで。時間の都合で全画面のクリック網羅はしていない。
- **a11y `snapshot -i`** … 本 SPA ではツリーがほぼ空に近く、操作は `eval` / `fill` / `find` で代替。

## 結論

- **本番**: ゲスト導線・ログイン失敗表示・保護ルートのリダイレクト・公開プロフィールの 404 メッセージは期待どおり。`/users/1` の「見つかりません」系はデータ要確認。
- **ローカル**: ログイン → ホーム → プロフィール往復 → リロードまでで、**以前の課題だった「ログイン済みなのに認証エラー表示」は再現せず**（文言ベース）。
